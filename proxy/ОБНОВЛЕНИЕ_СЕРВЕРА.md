# Пошаговый план обновления

Обновить нужно: **proxy на сервере** и **плагин в Figma**.

---

## Часть A. Обновление proxy на сервере

### Шаг 1. Узнать IP сервера

Откройте **cmd** (Пуск → cmd → Enter) и выполните:

```cmd
ping gemini-proxy.mooo.com
```

В первой строке ответа будет IP (например, `123.45.67.89`). Запишите его.

---

### Шаг 2. Перейти в папку proxy

```cmd
cd /d "c:\Users\Mikhail\Desktop\Check design\Check UX-design\proxy"
```

---

### Шаг 3. Скопировать файлы на сервер

Подставьте вместо `123.45.67.89` ваш IP из шага 1 и выполните:

```cmd
scp server.js mcp-server.js package.json package-lock.json root@123.45.67.89:/opt/gemini-proxy/
```

При запросе «Are you sure...» введите `yes`, затем пароль root.

---

### Шаг 4. Подключиться к серверу

```cmd
ssh root@123.45.67.89
```

(Подставьте свой IP.) Введите пароль root.

---

### Шаг 5. Обновить токен Figma (если нужно)

Если нужно обновить или добавить `FIGMA_ACCESS_TOKEN`:

```cmd
nano /opt/gemini-proxy/.env
```

- Если файл есть: найдите строку `FIGMA_ACCESS_TOKEN=...` и замените значение на новый токен.
- Если файла нет или он пустой: добавьте строку `FIGMA_ACCESS_TOKEN=figd_ваш_токен_здесь` (и при необходимости `GEMINI_API_KEY=...`, `PORT=3000`).

Сохранить: **Ctrl+O**, Enter. Выйти: **Ctrl+X**.

> Токен Figma: [figma.com](https://www.figma.com) → Settings → Account → Personal access tokens → Create new token

---

### Шаг 6. Команды на сервере (по одной)

```cmd
cd /opt/gemini-proxy
```

```cmd
npm install
```

```cmd
pm2 restart gemini-proxy
```

```cmd
exit
```

---

### Шаг 7. Проверка proxy

В браузере откройте:

https://gemini-proxy.mooo.com/metadata-components

Должен вернуться JSON с полем `componentKeys` (массив из 5 строк) и `fileKey`.

Если `componentKeys` пустой — проверьте `FIGMA_ACCESS_TOKEN` в `.env` на сервере.

## Часть B. Обновление плагина в Figma

### Шаг 8. Собрать плагин (если меняли код)

В cmd на вашем компьютере:

```cmd
cd /d "c:\Users\Mikhail\Desktop\Check design\Check UX-design"
```

```cmd
npm run build
```

### Шаг 9. Обновить плагин в Figma

1. Откройте Figma.
2. Правый клик по холсту → **Plugins** → **Development** → **Import plugin from manifest…**
3. Укажите файл: `c:\Users\Mikhail\Desktop\Check design\Check UX-design\manifest.json`
4. Если плагин уже добавлен — выберите **Plugins** → **Development** → ваш плагин → **Reload** (или закройте и откройте плагин заново).

### Шаг 10. Проверка плагина

1. Откройте файл с подключённой библиотекой Tools.
2. Запустите плагин.
3. Нажмите **«Оформить файл»**.
4. Должны появиться страницы Cover и Метаданные с компонентами из Tools.
